I / Foundations Primitives.

Contract

Every numeric value in any stylesheet — font-size, line-height, letter-spacing, color, spacing, radius, shadow — must be a var(--…) reference to a token defined in tokens.css. Raw rem, px, em, %, and hex values are not permitted outside the primitive layer. When a value is needed that does not exist, the primitive scale is extended; the system grows to absorb the requirement, and consumers continue to reference tokens only.

The token system is layered. Primitives (this page) define raw scale positions with no semantic role. Semantic tokens (Section 02) alias primitives and encode role. Component classes (e.g. .text-display, .h1) reference semantic tokens only. Each layer references strictly the layer below it.

Semantic role assignment (which step is body, which is h1, which is display) and component classes (.h1, .text-display, …) are defined in Section 01b — Typography components. Components must reference semantic tokens; semantic tokens reference these primitives.
