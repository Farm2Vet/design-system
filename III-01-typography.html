Section 01b / Foundations Typography components.

Contract

Every numeric value in any stylesheet — font-size, line-height, letter-spacing, color, spacing, radius, shadow — must be a var(--…) reference to a token defined in tokens.css. Raw rem, px, em, %, and hex values are not permitted outside the primitive layer. New sizes are introduced by extending the primitive scale; consumers continue to reference tokens only.

01 / Architecture

Three-layer model.

The typography system is structured in three strict layers. Each layer references the layer immediately below it; no layer skips down or sideways.

  1. Primitive scale — --text-xs through --text-9xl, defined in tokens.css. Thirteen steps on a strict 1.2 ratio anchored at --text-base = 1rem. The primitive layer carries no semantic role. See Section 01 — Primitives for the enumeration.
  2. Semantic tokens — --text-body, --text-eyebrow, --text-h1 through --text-h6, --text-display-sm, --text-display. Defined in tokens.css under SEMANTIC TOKENS. Each semantic token is a var() alias of a primitive. Renaming the role here updates every consumer.
  3. Component classes — .h1–.h6, .text-display family, .body, .body-sm, .caption, .eyebrow, .text-accent, .italic. Defined in typography.css. Components reference semantic tokens only. Pages apply component classes to markup.

Element-level CSS rules (h1 {}, h2 {}, p {}) declare only family, colour, and margin defaults. They do not declare font-size, weight, italic, or letter-spacing. A bare <h1> with no class therefore inherits no visible size; visual treatment is opt-in via component class.

03 / Components

Class previews.

Each component class below renders the role at its specified size, weight, line-height, and tracking. Apply the class directly to the appropriate semantic tag (e.g. <h2 class="h2">, <p class="body">). Display-tier classes (.text-display family) are used for hero and chunk-head contexts; the heading classes (.h1–.h6) are used for inline content.

04 / Heading levels

Semantic specification.

Heading rank is a semantic decision, independent of visual size. Choose the heading tag that correctly nests the page's document outline; choose the visual treatment separately by applying the appropriate component class.

<h1> — Page title
Exactly one per page. The page's primary heading; almost always the hero. Apply class="text-display" for the page hero treatment, or class="h1" when the page title appears inline within content.
<h2> — Top-level section
A major section within the page. In the section template, this is the chunk-head heading (Color, Spacing, Typography, Shadow). Apply class="text-display-sm" for the chunk-head treatment, or class="h2" for inline section headings.
<h3> — Subsection
A subdivision within a section: a card title, a named group (Primary, Secondary, Status), an inline content block. Apply class="h3".
<h4> — Labelled grouping
A grouping within a subsection: Weights, Line heights, Letter spacing. Apply class="h4" or class="subheading" when paired with the .tag pill.
<h5>, <h6> — Deep nesting
Reserved for content with nested structural depth (technical documentation, long-form articles). Rare in component-driven interfaces. Apply class="h5" or class="h6".
05 / Italic & emphasis

When to em, when to italic.

Italic in this system is governed by three distinct mechanisms. Each has a defined purpose; do not interchange them.

<em> — Stress emphasis
The semantic element for emphasis. Use when removing the italic would change the sentence's meaning (e.g. this is the one that matters). Renders italic by default. Inside a .text-display family heading, <em> additionally receives the brand voice colour (forest green for hero tier, earthy brown for chunk-head tier) by typographic component rule. The italic on display headings is documented as a decorative voice treatment and is not interpreted as stress emphasis by assistive technology in that context.
<i> — Alternate voice
Use for foreign-language phrases, technical or scientific names, ship names, internal monologue, and similar conventional italic uses where there is no stress meaning. Renders italic by default. Receives no automatic colour treatment. Reserved for editorial or content-rich pages.
.italic — Decorative italic
A utility class for italic with no semantic weight. Use when the visual italic is wanted but the content carries neither stress nor an alternate-voice convention. Apply to a <span>; do not apply to a heading or paragraph as a whole.

Element rules in base.css never apply italic. A bare <h2> renders upright. Italic appears only when one of the three mechanisms above is invoked.

Voice and emphasis colour.

Voice colour is an axis independent of italic. The utilities below shift colour without altering style or weight; combine with <em> or .italic when both treatments are intended.
