III-11 / Components Switch.

A binary on/off control. The thumb is the affordance — flipping it is the action; there is no Save step. The segmented variant handles one-of-N choices (language pickers, quick filters) with the same colour rail and the same height scale.

01 / Switch — Variants

Variants.

Two variants: the binary thumb-on-a-track for one true/false setting, and the segmented pill for picking one option from a short list. Both inherit the same height scale as Button and share the same on-state colour rail.

02 / Switch — States

States.

Four states: rest, hover, pressed, disabled. Hover and pressed cells use static data-state proxies — they paint the visual the live pseudo-class would produce, without altering behaviour. Focus is rendered by the standard outline ring on :focus-visible; it's a pseudo state rather than a separate cell here.

04 / Switch — Segmented

Segmented.

The segmented variant carries the same colour rail as the binary switch but renders the active selection as a sliding pill behind the chosen option. Two- and three-way groups cover the common shapes — language picker, caseload filter, locale chooser.

Use a switch for a binary on/off setting whose change takes effect immediately. No Save button, no confirmation step — flipping the thumb is the action. A switch is wrong for anything destructive or anything that needs explicit confirmation; reach for a button paired with a dialog there. Pair every switch with a label describing what it controls in the affirmative — "Notifications", not "Disable notifications" — so the on-position reads true. The visible label sits next to the switch; never let the slider stand alone.
