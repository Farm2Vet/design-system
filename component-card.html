Section 05 / Components Card.

The container the rest of the system lives inside. One shared shape adapts across three style variants, three layouts, four functional modes, four content patterns, a five-tier glass scale, and a full state matrix — all resolved through the --card-* token layer so a tweak to surface, border, or selected colour cascades to every card on the screen.

Architectural note · scope boundary

FAQ-style Q&A is the Accordion component's responsibility, not Card's. A question that expands to reveal an answer is a disclosure pattern with its own keyboard model, animation, and aria contract — different in kind from a content container. The Card system therefore does not ship a .card--question variant; the equivalent affordance lives under component-accordion.html (in a future release).

01 / Card — Anatomy

A shared structure.

Every variant is composed from the same set of slots — an optional media area, an eyebrow, a title, an optional subtitle, a body, and a footer with actions. Plus configurable sub-components: badges, prices, ratings, metrics, and avatars. Variants differ in which slots they expose, not in the underlying layout.

02 / Card — Style variants

Style by elevation.

Three style treatments along a single axis: how loudly the card announces itself against the canvas. Elevated is the workhorse default; Filled mutes it for secondary content; Outlined trades shadow for a confident border in dense layouts.

One style per region. Mixing Elevated and Outlined cards in the same grid creates competing depth cues. Pick the style that matches the region's elevation budget — marketing surfaces tend Elevated, dashboards and admin tooling tend Outlined or Filled.

03 / Card — Layout & orientation

Three orientations.

Vertical stacks media on top of content. Horizontal places media on the leading edge. Content-only drops media entirely. List bleeds rows to the card's edge for stacked content. Pick by viewport and scan direction, not by aesthetic preference.

06 / Card — Glass scale

Blur and transparency.

Five glass tiers, ordered by frostiness. Each pair of values (--card-glass-{tier}-blur and --card-glass-{tier}-bg) is designed in concert: more blur is paired with higher tint opacity, less blur with a clearer base. Backdrop is a soft pastel atmosphere, not a brand wash.

Pick the tier by what's behind the card. .card--glass-clear for floating overlays; .card--glass for hero callouts on subtle backdrops; .card--glass-frost for content surfaces over busy imagery; .card--glass-heavy for modal-grade slabs; .card--glass-tinted when the brand should colour the glass itself.

07 / Card — States

Five interaction states.

Default, Hover, Focus, Pressed, Disabled — applied automatically to any interactive card (an <a>, a <button>, or .is-interactive). The hover lift and pressed settle are the same micro-animations used by the Button component, kept under 200 ms so the UI never feels sluggish.

Default

Resting state.

Subtle shadow, default border.

Defaultrest
Hover

Lifted, brighter.

Tile lifts 1px; shadow deepens; border strengthens.

Hover:hover
Focus

Keyboard ring.

2px outline + 2px offset. Inherits the system focus colour.

Focus:focus-visible
Pressed

Settled in.

Tile drops back to baseline; bg darkens one step.

Pressed:active
Disabled

Not available.

Pointer-events suppressed; opacity reduced; no shadow.

Disabled.is-disabled
