III-15 / Components Table.

A grid of cells over tabular data. Use Table when columns are real columns and the user will sort or compare across rows. For activity feeds and free-form lists of mixed items, reach for Card · List instead.

Anatomy.

The component renders as a real <table>. A head, a body, head cells, rows, and cells — each addressable by a BEM-style class hook. Sortable head cells wrap their label in a .table__sort button so the entire label is the click target.

A · root
The wrapper (.table) is rendered as a real <table> element. Variant and size modifiers go on the root.
B · head
.table__head (<thead>) holds a single .table__head-row (<tr>) of .table__head-cell (<th>) cells. Background and eyebrow typography come from the component tokens.
C · sort
.table__sort is a button placed inside a head cell when the column is sortable. The whole label is the click target. Direction states are .is-asc and .is-desc; the icon flips on .is-desc.
D · body
.table__body (<tbody>) holds the .table__row (<tr>) instances. Rows divide on top borders so hover and selected backgrounds can paint without a seam.
E · cell
.table__cell (<td>) supports four modifiers: --mono (codenames, IDs), --meta (smaller, muted text for timestamps), --num (tabular-numeric, right-aligned), and --state (right-aligned badge slot).

Variants.

Two surface variants. Plain is borderless and lives inside a parent that owns its frame (a panel, a card-body). Boxed carries its own rounded border and lid; reach for it on V-01 and anywhere the table is a standalone region on the page.

Boxed by default on standalone tables. A bare .table with no frame floats — it reads as a fragment of a parent that doesn’t exist. Use --boxed unless the table is a child of a card or panel that already owns the border.

States.

Four row states stack in the same table: default, hover, selected, and dim. Hover paints a subtle row background; selected lifts the whole row to the primary-tinted surface; dim de-emphasises a row via opacity without changing its content. The sort affordance has three states of its own — idle, ascending, descending.

Sizes.

Two sizes. md is the default — comfortable row height, body-sm cell text. --sm compresses the row to a denser eyebrow-size text; reach for it in dense admin tables where many rows are visible at once.

Reach for Table when columns are stable and the user will sort or compare across rows. Use one badge per row in .table__cell--state — additional state flags belong in row-level treatments (.table__row--dim, .is-selected) or in a separate column. When a row carries a free-form mix of icon, title, description, and trailing meta with no shared schema across rows, that’s a Card · List, not a Table. Future work: an active filter / search state that swaps the toolbar label-row for a single-input search bar; rest-state only ships today.
