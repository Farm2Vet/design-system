III-13 / Components Chip.

A toggleable filter pill. Multi-select by contract — chips stack as additive filters within a row, OR-combining values on the same axis and AND-combining across axes. Composes a label, an optional leading slot for the on-state check, and an optional trailing count. Sits between .badge (read-only), .tabs (single-select view switcher) and .btn--toggle (single boolean) — none of which carry the right grammar for a stack of independent filter values.

01 / Chip — Scope

Scope.

Reach for a chip when the user needs to apply more than one filter at the same time, and each filter should stay visible after it's set. The chip is a toggle pill — clicking it stacks a value onto the active filter set; clicking it again removes that value. Selection is independent per chip.

If exactly one item from the set must be selected, use Tabs or a segmented switch instead. Chips have no concept of "the chosen one" — they only know "on" and "off" per chip. A row of chips with no chips active is a valid, meaningful state ("show everything"). A row of tabs with no tab selected is an authoring error.

Component Selection Reach for it when
.chip Multi-select toggle (independent on/off per chip). A list view exposes several axes of filtering and the user may want to stack values: status: Active OR Incomplete · species: Swine. Each chip is independent; no chips selected means "no filter applied."
.tabs Single-select view switcher. The user picks one of N peer views of the same content area. Selecting a tab mounts a panel; one tab is always selected.
.btn--toggle Single boolean (one chip, on or off). A single binary preference next to a control — "Show archived," "Compact rows." Use the button form when the toggle stands alone, not in a row.
.badge Read-only status marker. The element communicates state but is not interactive. Badge is what you reach for to label a row, not to filter one.
.checkbox-list Multi-select inside a form. The choice is part of a form being submitted, where label legibility and dense vertical scanning matter more than horizontal compactness.
02 / Chip — Anatomy

Anatomy.

A pill container with three optional slots: a leading icon (revealed only on the on-state), a required label, and a trailing count. Slots are addressable by BEM-style class hooks; the chip defaults to aria-pressed="false" and flips to "true" when toggled on.

A · container
.chip renders as a <button type="button"> with aria-pressed. The pill shape, fixed 28px height, and inline-flex layout are owned by the block — variants and modifiers compose by adding modifier classes, never by re-styling the block.
B · leading slot
.chip__leading holds the on-state icon (typically a check). The slot collapses visually at rest — authors can keep the markup in place; the chip's :not(.is-active) rule hides the slot until the chip toggles on.
C · label
.chip__label is the human-readable filter value. Required; never abbreviated to a glyph alone (use a leading icon if a glyph is needed alongside the label).
D · count
.chip__count is the optional read-only trailing number — "if you toggled this, how many rows would match?" Renders in tabular numerals so columns of chips stay aligned. Skip the count when it's not knowable, not stable, or not useful.

Removable is for the "active filters" tray, not the filter row itself. The toggle chip stays in the row — its × is implicit (click it again to turn it off). The removable chip leaves the tray when × is clicked. Mixing the two in the same row is an authoring error.

04 / Chip — States

States.

Seven discrete states. The on-state's tinted fill is the chip's primary affordance — the leading check is a reinforcement, not the signal — so the contrast between rest and active is the most important contract to preserve when retuning tokens.

Don't lean on the leading check to carry the on-state. The tinted fill, the primary border, and the medium-weight label are doing the work; the check icon is a reinforcement. If you skip the leading slot entirely, the chip still reads correctly. If you keep the slot but suppress the tint, the chip stops reading as "on" — that combination is an authoring error.

05 / Chip — In context

Filter row.

The pattern V-01 reaches for: a horizontal row of toggle chips with optional "Clear filters" affordance. Within a row, chips on the same axis OR together (status: Active or Incomplete); chips across axes AND together (status AND species). No chip selected = "show everything," with the Clear button hidden.
