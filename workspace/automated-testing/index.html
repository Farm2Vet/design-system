FR-384 / Test Suite Management

Lo-fi wireframes — automated testing console

Five screens covering UC-384 CRUD on test suites, FR-385 per-case metrics view, and FR-386 full-conversation deep dive. Grayscale only, no brand color or visual treatment — structure and information hierarchy only.

Author: Cowork (Claude) for Jack Status: design exploration · pre-spec Target route: /admin/testing

Screen 01 · /admin/testing/suites
Test suite list
List · filters · bulk actions
Sidebar nav, header w/ New-test-suite, filterbar (search · model · mode · status · sort), bulk action strip, dense table w/ status chips, pagination. UC-384 main flow.
Screen 02 · /admin/testing/suites/{id}
Test suite detail
Configuration · profiles · runs
Chips + actions (Run now), 6-cell metastrip, tabs (Configuration · Runs · Test cases · Activity), three sections: Configuration, Synthetic profiles (chips + list), Recent runs table.
Screen 03 · /admin/testing/suites/new
New / edit test suite
Create & update form
Four-section form: Identity, Model & mode, Synthetic profile picker (pool + selected, w/ disease filter chips), Execution (workers stepper, repetitions, anomaly + accuracy reports). Sticky save bar. UC-384 create/update.
Screen 04 · /admin/testing/suites/{id}/runs/{rid}
Run results
Per-case metrics table
Header w/ download report, 6-cell metastrip, 5 summary stat tiles (acc · flu · eff · lat · cost), download panel (.md / .json / anomalies / accuracy), case-status chips, dense per-case table w/ Flu · Eff · Lat · Turns · Acc · Cost · Step. FR-385.
Screen 04r · /admin/testing/suites/{id}/runs/{rid}
Run results (running)
Live progress · partial stats
Same shell as 04 in the running state. Pause/Cancel header actions, Progress metastrip cell, live segmented progress band, collapsed worker activity panel, striped stat tiles marked "so far", case table with 7 done · 2 in-flight · 3 pending rows.
Screen 04p · /admin/testing/suites/{id}/runs/{rid}
Run results (paused)
Resumable mid-run snapshot
Paused-state mirror of 04r. Resume/Discard header actions, paused tone (warning chip), "stopped mid-case" progress marker, idle workers, snapshot-only report download. Case table shows 4 done · 1 stopped mid-conversation · 7 pending.
Screen 06 · /admin/testing/executions
Executions list
Console for all runs
Admin-wide executions log. Filterable by suite/status/sort, dense table w/ state-aware status donut, bucket-mini outcomes, Time column with elapsed/ETA/paused/total shapes, per-state action trio (view results · download · remove).
Screen 05 · /admin/testing/suites/{id}/runs/{rid}/cases/{tcid}
Test case · full conversation
Conversation + scores + latency
Header, 6-cell metastrip, tabs, two-column body: chat history (SP vs bot) + side panel (scores · hypothesis evolution timeline · chatbot feedback), reasoning trace, latency breakdown table. FR-386.

Wireframe conventions

Grayscale rail only — no brand color, no shadow, no glass. Components shown as boxy approximations (.btn, .chip, .input, .table). Status uses chip variants (chip--ok / chip--run / chip--warn / chip--err / chip--idle). Metric values are illustrative — names, profile IDs, and numbers are placeholders for layout exploration only.
