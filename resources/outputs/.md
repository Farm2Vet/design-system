## 1. Structural sweep — Python script

Run the script below from the repo root (`D:\F2V\F2V Design System Final\design-system\` — bash sandbox path: `/sessions/vigilant-vibrant-fermat/mnt/design-system/`). It pattern-matches the canonical `chunk-head` shape, lifts the summary `<p>` out, and writes the file atomically via temp-rename so no NULL padding can be left behind. Files where no chunk-head matches are skipped.

```python
"""
Spec C — section-anatomy structural sweep.
Lift the summary <p> out of every <header class="chunk-head"> and re-emit it
as a <p class="section-lede"> sibling. Drop the inner <div> wrapper around
num-tag + h2. Atomic temp-rename per file to avoid trailing NULL padding
on the Windows mount.
"""

from __future__ import annotations
import re
from pathlib import Path

FILES = [
    "I-primitives.html",
    "II-semantics.html",
    "III-01-typography.html",
    "III-04-dropdowns.html",
    "III-05-cards.html",
    "III-06-modal.html",
    "III-08-toast.html",
    "III-09-alert-badges.html",
    "III-10-header-navigation.html",
    "III-12-sidebar.html",
    "III-14-tabs.html",
]

# Match the canonical chunk-head block:
#   <indent><header class="chunk-head[ reveal]">
#   <indent>  <div>
#   <indent>    <span class="num-tag">…</span>
#   <indent>    <h2 class="text-display-sm">…</h2>
#   <indent>  </div>
#   <indent>  <p>…</p>
#   <indent></header>
CHUNK_HEAD_RE = re.compile(
    r'(?P<indent>[ \t]*)'
    r'<header class="chunk-head(?P<reveal>(?: reveal)?)">'
    r'\s*<div>\s*'
    r'(?P<numtag><span class="num-tag">[^<]*</span>)'
    r'\s*'
    r'(?P<h2><h2 class="text-display-sm">.*?</h2>)'
    r'\s*</div>\s*'
    r'<p>(?P<summary>.*?)</p>'
    r'\s*</header>',
    flags=re.DOTALL,
)

def restructure_block(m: re.Match) -> str:
    indent = m.group("indent")
    reveal = m.group("reveal") or ""
    numtag = m.group("numtag").strip()
    h2 = m.group("h2").strip()
    summary = m.group("summary").strip()
    inner = indent + "  "
    return (
        f'{indent}<header class="chunk-head{reveal}">\n'
        f'{inner}{numtag}\n'
        f'{inner}{h2}\n'
        f'{indent}</header>\n\n'
        f'{indent}<p class="section-lede{reveal}">\n'
        f'{inner}{summary}\n'
        f'{indent}</p>'
    )

def sweep(path: Path) -> int:
    text = path.read_text(encoding="utf-8")
    new_text, n = CHUNK_HEAD_RE.subn(restructure_block, text)
    if n == 0:
        return 0
    tmp = path.with_suffix(path.suffix + ".tmp")
    tmp.write_text(new_text, encoding="utf-8")
    tmp.replace(path)  # atomic on the same volume
    return n

if __name__ == "__main__":
    total = 0
    for fname in FILES:
        p = Path(fname)
        if not p.exists():
            print(f"  missing:   {fname}")
            continue
        n = sweep(p)
        total += n
        status = f"{n:>3} chunks" if n else "no change"
        print(f"  {status}: {fname}")
    print(f"\n  total chunks restructured: {total}")
```

Expected output: ~99 chunks rewritten across the 11 files. Any file reporting "no change" deviates from the canonical pattern — pause and flag it; do not improvise an edit.

## Gotchas

1. **No `sed -i`.** The structural sweep uses Python with atomic temp-rename (Section 1's script does this correctly). Heading rewrites use the Edit tool. Either is fine; `sed -i` is not.
2. **Closing tags.** Each file ends with `</body>\n</html>\n` (some pages have a trailing `</script>` block before `</body>`). Confirm the bottom of each file is intact after the structural sweep — particularly for files with embedded scripts.
3. **`reveal` class preservation.** The structural sweep's regex captures the optional `reveal` class on the original `<header>` and re-applies it to both the new `<header>` and the new `<p class="section-lede">`. If a chunk-head originally had no `reveal` class, neither should the lede. Spot-check one file after the script runs to confirm.
4. **Fence-post scan.** After the script runs, grep `<header class="chunk-head"` across the 11 files — every match should now be a single-block header (no inner `<div>`). Any survivor means the chunk-head deviated from the canonical pattern and needs manual review.
5. **Heading rewrites are not regex-safe.** Two pages contain `<h2 class="text-display-sm">Token <em>map</em>.</h2>` (modal, sidebar, etc.) — apply each Edit per-file with the correct file_path; don't try to find-and-replace across the repo.
