III-22 / Components Menu.

Click/Escape-dismissed floating action panel. Semantically a role="menu" — not a form control. Unlike Dropdown, Menu owns no trigger shape; the call site supplies its own button. The panel reuses --dropdown-* surface tokens so the two floating surfaces read as one visual family.

Tokens

Menu defines 4 component tokens. All panel-surface and option-row geometry comes from shared --dropdown-* tokens — see III-04 Dropdowns.

Menu vs Dropdown. Use .dropdown when the trigger is a form control that displays the selected value (e.g. "Sort by: Name"). Use .menu when the trigger is an icon button or contextual action button that doesn't reflect the selection in its own label. Both share the same floating-panel visual language via --dropdown-* tokens.

Checkable items. Add .menu__item--checkable to reserve a 14 px leading slot for the check icon. The icon is always in the DOM — opacity:0 when not selected and opacity:1 when .is-selected is present. This keeps all item labels at the same indent regardless of selection state.
