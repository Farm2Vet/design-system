III-19 / Components Tooltip.

Frosted speech-bubble tooltip styled to the macOS HUD look. A thin near-white glass pane (44 % tint + sheen + blur) with a matching glass tail — both surfaces share the same material so they read as one continuous piece of glass, not a bubble stuck on top of a triangle.

The tooltip is self-contained (no Radix, no third-party popper). In production it renders via createPortal directly into <body> and positions with fixed top/left — never transform — so backdrop-filter keeps frosting the live page behind it.

Anatomy.

Three elements: the trigger wrapper that owns the hover target, the bubble body, and the tail triangle. Bubble and tail are siblings — not parent/child — so each can independently apply backdrop-filter and frost the same page content with identical material.

.tooltip
Trigger wrapper — position: relative; display: inline-flex. Owns :hover and :focus-within to show/hide the popup. Wrap any element: a button, icon, badge, or plain text.
.tooltip__popup
Absolute-positioned column container holding bubble + tail. visibility: hidden; opacity: 0 at rest; transitions to visible on hover. Add .is-open to force-show for documentation or testing.
.tooltip__bubble
The glass body: 44 % white tint, --blur-lg backdrop-filter, --glass-sheen-light sheen gradient, --shadow-md + --glass-inset-light-strong shadows. Max-width 260 px; add .tooltip__bubble--wrap for multi-line content.
.tooltip__tail
The glass triangle arrow: 16×8 px, clip-path: polygon for the direction. Same glass fill as the bubble (no shadow — a shadow on the triangle reads as detached). z-index: 2 so it floats in front of the bubble and the body's drop shadow doesn't dim it. Overlaps bubble by 1 px for a seamless join.

Placement.

Positioning constraint.

The frosted glass depends on backdrop-filter, which stops working when any ancestor element creates a backdrop root. The DS demo page is fine because no ancestor has a transform. Production requires extra care.

These CSS properties on any ancestor kill backdrop-filter by creating a backdrop root:

transform  ·  filter  ·  opacity < 1  ·  clip-path on a parent  ·  backdrop-filter on a parent

That is why the production <Tooltip> renders via createPortal directly into <body> and positions with fixed top/left computed by getBoundingClientRect — never transform: translate (the way Radix's popper works). Framer Motion animations on the page-level layout are the specific ancestor that motivated this design.

Do not replace the positioning logic with a Radix/Floating UI popper unless those libraries offer a transform: none mode.

Tokens.

All visual properties are overridable via CSS custom properties on :root. The glass recipe tokens (--tooltip-blur, --tooltip-saturate) map to the system glass rail in tokens.css so the tooltip tracks any global glass-recipe changes automatically.

Glass fill recipe — tint alpha is 44 % by design. Lower alpha is what lets the page genuinely refract through the glass; at higher alpha the frost reads as a flat near-opaque fill. Do not raise --tooltip-bg tint beyond 55 % without testing on a busy background.
