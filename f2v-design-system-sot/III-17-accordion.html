III-17 / Components Accordion.

A vertical stack of independent disclosures. Each item is a bordered card whose header is a real <button> carrying aria-expanded; clicking it reveals a body that animates open with the grid-template-rows: 0fr → 1fr technique — no magic max-height to overshoot or clip. Items are independent: any number can be open at once.

This is the one disclosure idiom the admin product reuses everywhere — the thought drawer, the judgment list, the follow-up editor all render this same trigger, a chevron that rotates 180° on open, and a grid-rows body. Reach for it to hide secondary detail behind a label; reach for .tabs when the sections are peer views the user switches between rather than detail they expand in place.

01 / Accordion — Scope

Scope.

Reach for an accordion when a surface carries secondary detail that should stay collapsed until asked for — a clinical record's sub-sections, a question editor's optional fields, a thought drawer's reasoning trace. The header is a real <button>; the body expands in place beneath it. Each item is independent, so multiple can be open at once.

Disclosure expands detail in place; tabs switch between peer views. If the sections are alternatives the user moves between — one visible at a time, always one selected — use .tabs. If they are detail the user expands alongside what's already on screen, and "nothing expanded" is a valid resting state, use the accordion.

02 / Accordion — Anatomy

Anatomy.

One item is a bordered card holding a header .accordion__trigger and an animated .accordion__panel. The trigger pairs a .accordion__title with a trailing .accordion__chevron; the panel wraps a clipping .accordion__body around the padded .accordion__content. The .is-open flag on the item flips the chevron and expands the panel's grid row.

Pyrexia (40.5–41.5 °C), anorexia, huddling, and reddened skin over the ears, abdomen, and inner thighs. Constipation followed by diarrhoea; some animals show a staggering, ataxic gait in the late acute phase.

A · item
.accordion__item is one disclosure — a bordered card (--border-default, --radius-md, --background-default). The .is-open state flag is the single source of truth: it flips the chevron and expands the panel. Items are independent, so several can carry .is-open at once.
B · trigger
.accordion__trigger is the full-width header <button> carrying aria-expanded. Transparent at rest, it tints to --background-row-hover on hover and shows a focus ring on :focus-visible. Left-aligned flex row — the title grows, the chevron pins to the trailing edge.
C · chevron
.accordion__chevron holds a ChevronDown svg that rotates 180° when the item opens (.accordion__item.is-open .accordion__chevron). It's a reinforcement of the open state, not the only signal — the expanded body carries that.
D · panel
.accordion__panel is the animated wrapper: display:grid; grid-template-rows:0fr, transitioning to 1fr on open. The 0fr → 1fr row resolves to the body's intrinsic height, so the panel grows to exactly the content height — no max-height guesswork.
E · body
.accordion__body is the clipping wrapper: min-height:0; overflow:hidden. It lets the grid row collapse below the content's min-content height and clips the content while the row animates. It wraps .accordion__content, which holds the padded inner copy.
03 / Accordion — Variants

Variants.

Three shapes share the same block. The default bordered card is the workhorse; the rail variant adds a coloured stripe for flagged or categorised items; the flush variant strips the card chrome to hairlines for dense inline-detail lists.

The rail tone is a flag, not the open-state signal. --rail-info, --rail-safe, and --rail-warning re-point only the 4px left border so an item reads as categorised (info / safe / warning) at a glance. They don't change the trigger fill or the open animation — the chevron and expanded body still carry the disclosure state.

04 / Accordion — States

States.

Five discrete states. Rest and open are the contract; hover and focus belong to the trigger; disabled mutes the trigger and drops the pointer affordance. The hover and focus cards below pin the visual statically — the live component produces them from :hover and :focus-visible.

The chevron reinforces the open state; it doesn't carry it alone. The expanded body and the aria-expanded="true" on the trigger are the real signals — the 180° chevron flip is a visual echo. If you suppress the body animation, keep aria-expanded truthful so assistive tech still reads the state.

05 / Accordion — In context

Disease record.

The pattern the conversation and question editors reach for: a record's secondary sections collapsed behind labels, expanded in place when the vet wants the detail. Try it — each item toggles independently, so you can keep several open while you read across.
