III-23 / Components Rich Text Editor.

WYSIWYG markdown field powered by TipTap v3. Toolbar is focus-reveal — collapses via grid-rows 0fr↔1fr with no row-height transition to avoid ratcheting on every editor transaction. Value round-trips as Markdown so the read-only Prose view renders identically to what was typed.

Anatomy.

Content formatting.

.rte__content styles ProseMirror output to match the read-only Prose view exactly. Headings, inline marks, lists, tables, images, links, and inline code are all covered. The WYSIWYG surface and the rendered view share the same CSS so bold looks the same whether editing or reading.

Implementation.

Key decisions behind the production MarkdownField component — quirks, constraints, and the rationale for each. Read this before modifying the RTE.

TipTap extensions

Production uses StarterKit (heading H1–H3, link, bullet list, ordered list, bold, italic, code, undo/redo) plus three explicit extensions. StarterKit's link is configured with autolink: false — see below.

  • @tiptap/starter-kit Heading H1–H3, lists, bold, italic, code, undo/redo, link (autolink off)
  • @tiptap/extension-image Image insertion with S3 presigned upload; renders <img> in the doc
  • @tiptap/extension-table Table + TableRow + TableHeader + TableCell — three separate packages
  • @tiptap/markdown Serialise/deserialise the doc as Markdown; enables editor.getMarkdown() and contentType: "markdown"
  • TableBoundaryGuard Local extension. Prevents the caret from getting trapped at the last cell of a table (ProseMirror's default behaviour).

Focus-reveal — no transition on grid-template-rows

The toolbar is shown/hidden by toggling grid-template-rows: 0fr ↔ 1fr. TipTap sets shouldRerenderOnTransaction: true so React re-renders on every editor transaction — including each keystroke. If grid-template-rows has a CSS transition, the browser re-resolves the 1fr track on every sub-pixel repaint, ratcheting the field taller character-by-character. Only the inner opacity is transitioned — it's safe because opacity never triggers layout.

{/* toolbar wrap — grid reveal, NO row transition */}
<div className={cn(
  "grid",
  showToolbar ? "grid-rows-[1fr]" : "grid-rows-[0fr]",
)}>
  {/* clip: overflow-hidden → overflow-visible when popper open */}
  <div className={cn(
    "min-h-0",
    panel === null ? "overflow-hidden" : "overflow-visible",
  )}>
    <div className={cn(
      "flex flex-wrap items-center gap-[2px] …",
      "[transition:opacity_0.18s_ease]",           /* ← opacity only */
      showToolbar ? "opacity-100" : "opacity-0",
    )}>
      {/* buttons … */}
    </div>
  </div>
</div>

shouldRerenderOnTransaction: true

TipTap v3 defaults this off for performance. We turn it on so that Bold / Italic / Underline and the block/link indicators in the toolbar reflect the actual mark state the instant a button is clicked — not only after the next keystroke pushes an onUpdate. These single fields are cheap enough that the per-transaction re-render is not a problem in practice.

const editor = useEditor({
  immediatelyRender: false,
  // Re-render toolbar on every transaction (selection + stored-
  // mark changes) so active indicators stay in sync immediately.
  shouldRerenderOnTransaction: true,
  extensions: [ … ],
  …
});

External value sync — guard against editor.isFocused

When the controlled value prop changes externally (language switch, Discard), TipTap's setContent rebuilds the whole document and resets the cursor. During editing, the controlled value briefly lags the edit because onUpdate is async. To avoid the caret jumping on every keystroke, we only reconcile when the editor is not focused, and only when the current markdown diverges from the incoming value.

useEffect(() => {
  if (!editor || editor.isFocused) return;   // ← key guard
  let current: string;
  try   { current = editor.getMarkdown(); }
  catch { current = editor.getText();    }
  if (value === current) return;             // already in sync
  editor.commands.setContent(value, {
    contentType: "markdown",
    emitUpdate: false,                       // don't fire onUpdate
  });
}, [value, editor]);

Link extension — autolink: false

StarterKit's link extension derives its mark's inclusive property from the autolink option. With autolink: true (the default), the link mark is inclusive — typing at a link's right edge extends the link. With autolink: false, the mark is non-inclusive: the caret escapes to plain text after the link. We also want links added via the toolbar dialog only, not auto-detected from typed URLs.

StarterKit.configure({
  heading: { levels: [1, 2, 3] },
  link: {
    openOnClick: false,
    autolink: false,   // non-inclusive mark + no auto-detection
  },
}),

Table — table-layout: fixed is required

Without table-layout: fixed, the browser uses automatic table layout. Typing into a cell widens that column character-by-character (the browser sees each keystroke as a layout change). Fixed layout sizes columns from the table's declared width; content wraps within the cell via word-break: break-words instead of pushing the column wider.

/* In .rte__content */
table {
  width: 100%;
  table-layout: fixed;    /* ← required; auto layout ratchets */
  border-collapse: collapse;
}
td, th { word-break: break-word; }

Toolbar buttons — onMouseDown: preventDefault

Without this, clicking a toolbar button blurs the editor before the onClick fires. The editor's blur handler collapses the toolbar, the selection is cleared, and execCommand / TipTap chain has nothing to act on. preventDefault on mousedown stops the focus transfer while still allowing the subsequent click to register.

export function TBtn({ title, onClick, … }) {
  return (
    <button
      type="button"
      onMouseDown={(e) => e.preventDefault()}  // ← key line
      onClick={onClick}
      …
    >
      {children}
    </button>
  );
}

Placeholder — doc-aware empty detection

TipTap's editor.isEmpty is text-based: inserting a table or a list with no text would leave isEmpty === true while the placeholder overlaps the block. Production checks the ProseMirror doc structure directly: exactly one child, that child is a paragraph, and the paragraph's content size is zero.

const doc = editor.state.doc;
const isDocEmpty =
  doc.childCount === 1 &&
  doc.firstChild?.type.name === "paragraph" &&
  (doc.firstChild?.content.size ?? 0) === 0;

{placeholder && isDocEmpty && (
  <p className="pointer-events-none absolute … italic …">
    {placeholder}
  </p>
)}

Tokens.

Image and table commands require TipTap extensions. The Image button triggers a file input that POSTs to /api/v1/files/upload, gets a presigned S3 URL, uploads directly to S3, then inserts the CDN URL via editor.chain().setImage(). This is not reproducible with document.execCommand. Similarly, table insertion requires @tiptap/extension-table. Both features are documented here as CSS/token contracts; the JS wiring belongs in the React component.

Tall variant. Add .rte__content--tall to increase the minimum editing height from --rte-content-min-h (160 px) to --rte-content-min-h-tall (320 px). Used for longer-form fields: disease overview, conversation scripts, supporting points.
