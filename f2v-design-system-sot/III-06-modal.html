III-06 / Components Modal dialog.

One pattern: a centered dialog for a single focused decision — a tone-mapped glass rim over a blurred scrim, spring in, fade out. (Drawers, sheets and size tiers were retired; a dialog is for deciding, not browsing.)

01 / Modal — Anatomy

Anatomy.

Shown open below — or to see it as a real overlay (spring-in; Escape or click-outside to close).

Parts: .modal (scrim host) → .modal__scrim · .modal__surface (the rimmed sheet) → .modal__close · .modal__header (.modal__icon · .modal__title · .modal__sub) · .modal__body · .modal__footer. Open with .is-open on .modal.

02 / Modal — Tones

Tones.

Only the rim changes. Info (white, default) for routine confirms; danger (critical) for destructive actions; cta (lime→forest gradient) for the one primary-action moment.

Delete 4 disease records?

This can't be undone.

The selected records and their English & Vietnamese content are permanently removed.

Run the full test suite?

312 cases · ~4 min.

Every synthetic profile is evaluated against the live model. You'll get a scored report when it finishes.
03 / Modal — Destructive

Destructive flow.

Two stages: an outline-danger trigger (.btn--caution-ghost, label ends in “…”) opens the danger dialog above; the dialog's confirm is the solid danger button (.btn--caution). Never destroy on the first click.
