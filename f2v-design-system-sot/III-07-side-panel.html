III-07 / Components Side panel.

A glass drawer that slides in from the edge over a blurred scrim — for a detail or edit view beside a list, or a filter panel, without leaving the page. Its own component now (no longer a modal variant).

01 / Side panel — Anatomy

Anatomy.

Shown open below — or to see it slide in as a real overlay (Escape or click-outside to close).

Disease · PRRS

Edit clinical details

Open beside the record so the list stays in view. The body scrolls on its own while the header and footer stay pinned — long forms and detail views fit without a full-page route.

Pair it with the canonical form, table, and badge components; the panel only owns the chrome (scrim, slide, edges).

Reset

Parts: .side-panel (scrim host) → .side-panel__scrim · .side-panel__surface (the drawer) → .side-panel__header (.side-panel__eyebrow · .side-panel__title · .side-panel__close) · .side-panel__body (scrolls) · .side-panel__footer (.side-panel__footer-aux for a left link). Open with .is-open; add .side-panel--left to anchor left.

Use a side panel to edit or inspect in context — the list stays beside it. Use a modal instead when the user must decide one thing before doing anything else.
