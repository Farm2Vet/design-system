III-21 / Components Dropzone.

Drag-and-drop file upload surface. Dashed border tints primary green while a file hovers over it. Click opens a hidden <input type="file">. Single-file: the first dropped or selected file is emitted via a callback.

Anatomy

Drop a file here or browse from your computer PNG, JPG, PDF up to 10 MB
.dropzone
Outer container. role="button" tabindex="0" makes it keyboard-accessible. JS attaches dragover, dragleave, drop, and click.
.dropzone__icon
Circular icon holder. Use a 24×24 stroke SVG (upload-cloud, file-plus, etc.).
.dropzone__title
Primary copy — semibold, --dropzone-title-fg.
.dropzone__browse
Inline span inside .dropzone__hint. Renders in --dropzone-browse-fg (primary strong) as a visual affordance for clicking.
.dropzone__meta
Optional format / size hint. Dimmed at --dropzone-hint-fg.
.dropzone__input
Hidden <input type="file">. JS triggers .click() when the surface is activated.

Tokens

Single-file contract. The dropzone emits the first file from the drop or selection; multi-file queuing is the page's concern, not the component's. Always provide a title that names the expected content (e.g. "Drop a profile image here") and set accept on the hidden input to constrain the file picker.
