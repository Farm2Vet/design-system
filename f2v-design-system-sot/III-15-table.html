Anatomy.

The .list-table is a flex-column card shell with four named zones. The header and footer share the same frosted glass surface formula. The header holds a separate <table> (not a sticky <thead>) so the glass rail can bleed to the card edges without being clipped by overflow:hidden.

.list-table
Card shell. flex-direction: column; overflow: hidden; outline border; border-radius.
.list-table__head
Frosted glass header zone. Contains a table--fixed header-only table. Set aria-hidden="true" — the body table carries accessible column headers in a visually-hidden <thead>.
.list-table__bulk + .list-bulk-bar
Animated slot. display: none by default; display: block on .is-open (JS adds this only when selection count > 0). The .list-bulk-bar inside carries the lime-to-forest gradient glass surface. Note: backdrop-filter ignores CSS overflow/grid clipping, so display:none is the only reliable hide mechanism.
.list-table__scroll
overflow-y: auto; flex: 1 1 auto. Contains the body-only table. overscroll-behavior-y: none prevents page scroll chain-through on iOS.
.list-table__footer
Frosted glass footer rail, fixed at 40 px. Holds the showing-count, .pager, and .rows-select. Same glass formula as the header.

Row states.

Hover and selection paint background on the <tr>. Dim rows de-emphasize inactive records — apply .table__row--dim to the row to reduce its opacity to 55%.

Sub-components.

The BulkBar and Pager are used inside .list-table but can also appear standalone. The Pager sits outside the list-table in any context where pagination controls are needed without the full card.

Use .list-table for every admin data-list screen — it is the only sanctioned table shell. Reach for the bare .table.table--boxed only when the content genuinely does not need pagination, bulk actions, or a scrollable body (e.g. a small inline reference table in a detail panel). Never hand-roll a sticky-thead workaround; use the two-table pattern (separate header table + body table with matching <colgroup>) so the glass rail can bleed edge-to-edge without overflow clipping.
