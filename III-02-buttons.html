05 / Button — Padding ratio

Padding ratio.

Buttons obey the golden band on both axes across every size: pad-y 0.5 – 0.75 of the label font, pad-x 1.0 – 1.5 of it. Unlike form inputs (which cap X at 32px above LG because the value floats in an empty rail), buttons keep scaling X with font — the label is the surface, so the padding has to grow with the word. SM is the density tier and sits below the Y floor by design, the same convention every peer system follows for compact buttons. See form fields for the capped sibling rule.

Size font pad-y pad-x H : V pad-x / font pad-y / font Control rail token
SM 13.33px 4 px 16 px 4.0 : 1 1.20 0.30 --space-control-x-sm
MD 16px 8 px 20 px 2.5 : 1 1.25 0.50 --space-control-x-md
LG 22.5px 12 px 32 px 2.67 : 1 1.42 0.53 --space-control-x-lg
XL 27.65px 16 px 40 px 2.5 : 1 1.45 0.58 --space-control-x-xl
2XL 33.18px 20 px 48 px 2.4 : 1 1.45 0.60 --space-control-x-2xl
Golden band — button rail:
  • pad-y / font ∈ [0.50, 0.75] at MD and above
  • pad-x / font ∈ [1.00, 1.50] at all sizes — no cap
  • SM is the density tier — intentionally below the Y floor

Form inputs share these tokens but cap pad-x at 32px above LG (the placeholder rail problem). See form fields for the capped sibling rule.

Content surfaces (alerts, toasts) run a separate, looser band — they hold multi-line prose, where vertical headroom matters more than horizontal trim. See alerts & badges.
