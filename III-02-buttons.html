00 / Button — Anatomy

Anatomy.

Six conceptual slots make up a button: the container, its surface and border, an optional icon, the label, and the keyboard focus ring. Slot names are descriptive — they're how we talk about the parts, not classes you compose. The Switch component has its own anatomy, documented in its own section.

# Slot What it is
1Container The outer button shape — sets size, padding, and radius.
2Surface The background fill, controlled by the variant.
3Border The outline, also variant-controlled (transparent on filled variants).
4Icon An optional leading glyph that supports the label.
5Label The action's verb — short, descriptive text.
6Focus ring The keyboard-focus outline drawn outside the button.

05 / Button — Switch

Use a switch for a binary on/off setting whose change takes effect immediately. No Save button, no confirmation step — flipping the thumb is the action. A switch is wrong for anything destructive or anything that needs explicit confirmation; reach for a button paired with a dialog there. Pair every switch with a label describing what it controls in the affirmative — "Notifications", not "Disable notifications" — so the on-position reads true. The visible label sits next to the switch; never let the slider stand alone.

06 / Button — Usage

Usage.

Six rules that keep buttons predictable across surfaces. Each captures the one decision that's easy to get wrong — label tone, emphasis, availability, hit target, destructive confirmation, glass context.

01

Labels are descriptive verbs.

Say what the action does. "Save draft", not "OK"; "Delete sample", not "Yes". No terminal punctuation — buttons aren't sentences.

02

One primary per surface.

At most one .btn--cta per visible region; the rest are .btn--secondary, .btn--tinted, or .btn--ghost. Two CTAs side-by-side leaves the user nowhere to look first.

03

Disable, don't hide.

When an action isn't currently available, render it disabled with [disabled]. Hiding the button removes the affordance the user already expected.

04

Hit target ≥ 44px.

.btn--lg is 48px tall — comfortably above the 44px minimum for reliable touch. .btn--md (36px) and .btn--sm (28px) are desktop-only or dense-table contexts; pair with adequate spacing.

05

Caution requires confirmation.

.btn--caution initiates a destructive action. Pair with a confirm dialog or undo affordance — never let a single click cause irreversible loss.

06

Glass needs content behind it.

.btn--glass exists for floating contexts — cards, hero overlays, side panels with imagery. On a flat surface it reads as dead-translucent; reach for .btn--secondary instead.
